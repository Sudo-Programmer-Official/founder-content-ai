# **9\. Database recommendation**

Use **Postgres**.

Why:

* relational structure fits well

* JSONB helps for flexible channel payloads

* good for analytics rollups early

* can use pgvector later if needed

Use JSONB for:

* AI metadata

* platform-specific payloads

* generation settings

* event metadata

But do not overuse JSONB for core entities.

Keep core fields structured.

prompts/

  linkedin/

  repurpose/  
  optimization/

Each prompt should include:

* output schema
